Doctors вЂ” view full doctor list with type/unit/phone. Create new doctor. Edit doctor details. Delete doctor (only if no visits/roster linked).

RECEPTIONIST

Dashboard вЂ” view: registered patients count, tokens issued today, active health cards count. Quick-action buttons to jump to register/token/card pages.
Patients вЂ” view patient list with search by name/ID/category. Create: register new patient (fill full form including conditional guardian for FAMILY). Edit: update patient phone/email/address. No delete.
Health Cards вЂ” view all cards with status and expiry. Create: issue a new card for a patient who doesn't have one (pick patient, set expiry, mark photo submitted). Edit: suspend / activate / expire a card.
Token Queue вЂ” view today's token queue grouped by unit, with status badges (WAITING / SERVED / CANCELLED). Create: issue a new token (select patient + unit; server validates active card and computes next token number). No edit or delete on tokens.
Duty Roster вЂ” view only (check which doctor is on duty in which unit/shift today). No create/edit/delete.
Ambulance вЂ” view ambulance live status (in service, on trip, maintenance). Create: log a new ambulance dispatch (select ambulance, optional patient, destination, trip type, requested by).


DOCTOR

Dashboard вЂ” view: waiting count in own unit, patients seen today, prescriptions written today. Shows the live token queue below.
Token Queue вЂ” view today's WAITING tokens for their assigned unit only. Each token has a "Start visit" button. No create/edit/delete on tokens.
Visits вЂ” view list of own visits (ones they conducted). Create: start a new visit from a WAITING token (form: vitals, complaint, diagnosis) OR create an emergency visit without a token (visit_type=EMERGENCY, select patient directly). Edit: update diagnosis / follow-up date on a past visit. No delete.
Prescriptions вЂ” view own prescriptions. Create: write a new prescription linked to a visit (advice, next visit date, and repeatable item rows: medicine, dosage, duration, quantity, instruction вЂ” add/remove items). No edit after creation (prescriptions are final). No delete.
Test Orders вЂ” view tests they ordered with status and results. Create: order a new diagnostic test for a patient (select test from catalogue). No edit on orders (lab tech enters results). No delete.
Admissions вЂ” view current bed/ward status and admitted patients. Create: admit a patient to the isolation ward (select patient, enter disease вЂ” bed auto-assigned by server). Edit: discharge a patient (set discharge_datetime + status=DISCHARGED). No delete.


PHARMACIST

Dashboard вЂ” view: total medicines count, low-stock alert count, today's dispensary revenue. Shows the dispense queue below.
Dispense Queue вЂ” view list of prescriptions that haven't been fully dispensed. Each shows patient name, doctor, date, and item details. Create: dispense medicine (for each prescription item, enter dispensed quantity; server runs the transaction вЂ” stock check, charge computation, stock decrement). No edit after dispense (dispense records are final). No delete.
Medicine Stock вЂ” view full inventory table (name, generic, form, strength, price, stock, reorder level, status). Create: add a new medicine to inventory. Edit: update stock quantity, unit price, reorder level. Delete: only medicines that have never been prescribed.
Low Stock Alerts вЂ” view medicines where stock < reorder level (highlighted in red). This is a filtered view, not a separate table.
Dispense Log вЂ” view all past dispenses: patient, category, items dispensed, amount charged (Free for students/homeo, cost price for teachers/staff), dispensed by whom. Read-only.


LAB TECH

Dashboard вЂ” view: pending test count, completed test count today, total catalogue size.
Pending Tests вЂ” view test orders with status ORDERED or SAMPLE_COLLECTED. Each shows patient, test name, category, ordering doctor, date. No create вЂ” doctors order tests.
Enter Results вЂ” edit a test order: mark sample_collected_at (datetime), change status to SAMPLE_COLLECTED or COMPLETED, enter result_value, set result_date, add remarks. This is the core action вЂ” updating the test order, not creating a new record. No delete.
Test Catalogue вЂ” view the diagnostic test list (test name, category, sample type, normal range, available days). Read-only for lab tech вЂ” admin manages the catalogue.


PATIENT (read-only on everything, own data only)

Overview вЂ” view: own profile (name, category, department, university ID), health card status and expiry, summary counts (visits, prescriptions, tests).
Visit History вЂ” view own visits only: date, doctor name, visit type, complaint, diagnosis, vitals (BP, temperature, weight, pulse), follow-up date. No create/edit/delete.
Prescriptions вЂ” view own prescriptions only: date, prescribing doctor, advice, medicine items with dosage/duration/quantity/instruction. No create/edit/delete.
Test Results вЂ” view own test orders only: test name, category, status, result value (when completed), date. No create/edit/delete.
Health Card вЂ” view own card: card number, issue date, expiry date, status (ACTIVE/EXPIRED/SUSPENDED). Visual card display. No edit вЂ” receptionist manages cards.
